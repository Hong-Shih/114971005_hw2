## C-2：質性分析與個人體會

### 1. 🔍 方法特性與個人感受

這次專案對比讓我們深入理解了文本處理領域的演變。

#### 傳統方法（TF-IDF）：傳統穩定的工具

* **優勢：**
    * **高效率、低成本：** 計算邏輯基於詞頻統計，速度極快，成本幾乎為零，適合大規模資料集建立索引或粗略匹配。
* **限制：**
    * **缺乏語義理解：** 無法判斷使用不同詞彙但意思相近的句子，在處理需要語義理解的任務時感到非常受限。

#### AI 方法（Gemini 2.5 Pro）：創新具生產力的工具

* **優勢：**
    * **革命性的理解與生成能力：** 尤其在文本摘要中，能抓住文章核心並用通順流暢的句子重新表達；可不經訓練處理全新的分類任務。
* **限制：**
    * **延遲與費用：** API 呼叫的延遲和費用是必須考量的成本，不適合隨心所欲地進行大量運算。

> **🎯 各自的適用場景：**
> 如果目標是「**快、省、篩選**」，選擇 **TF-IDF**；如果目標是「**準確、理解、生成**」，選擇 **Gemini 2.5 Pro**。

### 2. 💡 實作心得與個人體會

#### 實作過程中的主要挑戰

* **TF-IDF 的挑戰：資料預處理**
    * 挑戰在於「資料預處理」，需花費大量精力調整停用詞列表，本質上是在修補底層邏輯（不懂語義）的缺陷。
* **Gemini 2.5 Pro 的挑戰：提示詞設計 (Prompt Engineering)**
    * 挑戰在於「提示詞設計」，指令的微小變動會導致摘要風格或分類準確率產生巨大差異。學會與強大模型「對話」是關鍵。

#### 對兩種方法的理解和感受

| 方法 | 理解與感受 |
| :--- | :--- |
| **TF-IDF** | 像是 NLP 領域的「**數學基礎**」，教會我們從數據統計的角度看待文本，是紮實的根基。 |
| **Gemini 2.5 Pro** | 像是「**智能工具**」，讓我們能站在更高維度上解決問題，將精力從底層特徵工程轉移到上層應用設計。 |

#### 未來的學習方向

1.  **提示工程的精進：** 專注於開發更穩定、更高效的 Prompt 模板。
2.  **RAG (檢索增強生成) 的應用：** 學習如何結合傳統檢索（如 TF-IDF 或向量搜索）提供最新資料給 LLM 進行生成，解決模型知識滯後問題。

### 3. ⚖️ 應用建議：找到平衡點（粗篩 + 精煉策略）

這兩種方法不應被視為競爭者，而是**互補的工具**。

* **何時使用傳統方法？**
    * 當資料量達數十萬行，或系統預算緊張時，先用 **TF-IDF 快速過濾和篩選**，找出潛在的目標文檔。
* **何時使用 AI？**
    * 當需要對篩選出來的少量關鍵文檔進行**精細化處理**時（如分析客戶郵件的不滿、生成會議行動項目摘要）。

#### 最佳實踐：採用「粗篩 + 精煉」的策略

1.  **粗篩：** 使用 **TF-IDF** 或其他輕量級方法作為初級過濾器，將數據範圍從「大」縮小到「小」。
2.  **精煉：** 只將這個「小」範圍的數據提交給 **Gemini 2.5 Pro** 進行最終、高品質的語義處理。

> **效益：** 既利用了傳統方法的效率來**節省成本和時間**，又利用了 AI 的智能來**保證最終輸出的品質**。
